# 📋 YouTube 자동화 시스템 기획서 v1

## 1. 프로젝트 개요

### 1.1 목표
- **1차 목표**: AI 엔지니어 포트폴리오
- **최종 목표**: 수익화 (조회수, 광고)

### 1.2 핵심 철학
> "Config만 바꾸면 새 채널을 찍어낼 수 있는 시스템"

- 채널, 페르소나, 주제 범위 모두 설정 파일로 관리
- 코드 수정 없이 새로운 채널 런칭 가능
- 유튜브 알고리즘 최적화: 1채널 1주제 권장, 멀티 채널 운영

---

## 2. 채널 운영 전략

### 2.1 채널 구조
```
시스템
├── 채널 A (테크)
│   ├── 페르소나 A
│   ├── 주제 설정 (AI, 개발, 스타트업...)
│   └── 소스 설정 (해외 위주)
│
├── 채널 B (엔터)
│   ├── 페르소나 B
│   ├── 주제 설정 (예능, 드라마, 밈...)
│   └── 소스 설정 (국내 위주)
│
└── 채널 C (재테크)
    ├── 페르소나 C
    └── ...
```

### 2.2 채널 간 규칙
| 항목 | 정책 |
|------|------|
| 같은 주제 중복 | ✅ 허용 (다른 관점으로 다룸) |
| 주제 혼합 | ✅ 자유롭게 (AI+투자 → AI 관련주) |
| 채널당 주제 수 | 채널별 설정 (1개 집중 or 멀티) |

### 2.3 초기 계획
- **Phase 1**: 테크 채널 1개로 시작 (검수 편의)
- **Phase 2**: 검증 후 다른 분야 채널 확장

## 3. 콘텐츠 전략

### 3.1 영상 포맷
| Phase | 포맷 | 비고 |
|-------|------|------|
| 초기 | Shorts only (60초 이하) | 알고리즘 태우기 |
| 확장 | Shorts + Long-form | 수익화 극대화 |

### 3.2 영상 스타일
```
[자막] + [배경 이미지/영상] + [TTS 음성]
```
- **자막**: 필수, 핵심 강조
- **배경**: 스톡 영상/이미지 또는 AI 생성
- **음성**: TTS 필수 (무음 비선호, 트렌드)

### 3.3 업로드 빈도
- **최소**: 하루 1개 (유튜브 알고리즘)
- **목표**: 채널별 config로 설정 (하루 2-3개까지)

### 3.4 플랫폼
| 플랫폼 | 우선순위 | 비고 |
|--------|----------|------|
| YouTube | 🔴 당장 | 메인 |
| TikTok | 🟡 고려 | 같은 영상 재활용 |
| Instagram Reels | 🟡 고려 | 같은 영상 재활용 |

## 4. 콘텐츠 소스

### 4.1 소스 유형
```
국내 커뮤니티
├── 메이저: 클리앙, 루리웹, 인벤, 에펨코리아
```

### 4.2 소스 비중

- 국내 예능 → 국내 커뮤니티 위주

## 5. 페르소나 시스템

### 5.1 페르소나 컨셉
> **브랜드/채널 자체가 화자** ("~입니다" 스타일)

- 실제 사람 느낌 X
- 캐릭터/가상 인물 X
- 채널 = 화자 O

### 5.2 페르소나 구성 요소
```yaml
페르소나:
  identity:
    name: "채널명"
    tagline: "한 줄 소개"

  voice:
    gender: "male" | "female"
    voiceId: "TTS 음성 ID"
    tone: "친근함" | "전문적" | "유머러스"

  communication:
    style: "반말" | "존댓말" | "혼합"
    speechPatterns: ["~거든요", "~인 거죠"]
    avoidWords: ["혁신적인", "패러다임"]

  perspective:
    coreValues: ["실용성", "솔직함"]
    biases: ["과장 싫어함", "검증된 것 선호"]

  examples:
    - topic: "새 AI 도구"
      goodResponse: "..."
      badResponse: "..."
```

### 5.3 TTS 설정
- **성별**: 페르소나에 따라 결정
- **음성 종류**: 같은 성별도 다양한 보이스 중 선택
- **서비스**: 저비용 우선 (Edge TTS → ElevenLabs)

## 6. 주제 설정 시스템

### 6.1 주제 범위 설정
```yaml
topicConfig:
  # 찾을 것
  include:
    categories:
      - name: "ai"
        weight: 1.0
        subcategories: ["llm", "image-gen", "robotics"]
      - name: "startup"
        weight: 0.8

    keywords:
      - keyword: "ChatGPT"
        weight: 1.5
        synonyms: ["챗지피티", "GPT"]

  # 피할 것
  exclude:
    categories: ["politics", "religion"]
    keywords: ["대통령", "국회", "교회"]

  # 조건부
  conditional:
    - name: "테크 규제는 허용"
      condition: { keywords: ["AI", "규제"], operator: "AND" }
      action: "review"
```

### 6.2 주제 혼합
- **자유롭게 허용**
- 예: AI + 투자 → "AI 관련주 분석"
- 예: 게임 + 밈 → "커뮤니티 반응 정리"
- 시스템이 자연스러운 연결고리 찾아서 제안

## 8. 자동화 범위

### 8.1 핵심 파이프라인
| 단계 | 자동화 | 검수 |
|------|--------|------|
| 주제 수집 | ✅ 자동 | 초기 검수 |
| 스크립트 생성 | ✅ 자동 | 초기 검수 |
| 영상 제작 | ✅ 자동 | 초기 검수 |
| 썸네일 생성 | ✅ 자동 (AI) | - |
| 업로드 | ✅ 자동 | 초기 검수 |
| 메타데이터 | ✅ 자동 | - |

### 8.2 커뮤니티 관리
| 기능 | 범위 | 비고 |
|------|------|------|
| 댓글 모니터링 | ✅ 포함 | 부정 댓글 알림 |
| 댓글 자동 응답 | 🟡 검토 | 리스크 있음 |
| 커뮤니티 탭 | ✅ 포함 | 자동 포스팅 |

### 8.3 점진적 자동화
```
Phase 1: 모든 단계 수동 검수
Phase 2: 주제/스크립트 자동, 영상/업로드 검수
Phase 3: 업로드만 검수
Phase 4: 완전 자동화
```

## 10. 기술 스택 (확정)

### 10.1 핵심 스택 - Python 단일
```
Backend:      FastAPI (Python)
Database:     PostgreSQL + Redis
Vector DB:    Chroma (개발) → Pinecone (운영)
Queue:        Celery + Redis (or RQ)
Scheduler:    APScheduler (or Celery Beat)
```

### 10.2 AI/ML 스택
```
Embedding:    BGE-M3 (HuggingFace)
Retrieval:    Hybrid (Semantic + BM25)
Reranking:    BGE-Reranker
LLM (RAG):    Claude API (LangChain)
LLM (FT):     Llama 3 + LoRA (PEFT)
```

### 10.3 수집 스택
```
크롤링:       Scrapy + Playwright
RSS:          feedparser
API:          httpx (async)
```

### 10.4 영상 스택
```
TTS:          edge-tts (무료) → ElevenLabs (유료)
영상 합성:    FFmpeg (python-ffmpeg or subprocess)
자막:         pysrt + Whisper
이미지:       Pillow + DALL-E API
```

### 10.5 웹/대시보드
```
API:          FastAPI
Frontend:     React (or HTMX for simplicity)
Auth:         FastAPI + JWT
```

### 10.6 왜 Python 단일 스택?
- AI 엔지니어 포트폴리오 목적에 부합
- HuggingFace, LangChain 등 AI 생태계가 Python 중심
- 파인튜닝(LoRA, PEFT)은 Python only
- 새 모델/도구 나오면 Python SDK가 먼저 지원됨

## 12. 확정 사항 (전체)

### 12.1 콘텐츠 전략
- ✅ **시리즈 전략**: 시스템 자동 판단 (성과 기반 동적 시리즈화)
  - 같은 주제가 연속으로 잘 되면 자동 시리즈화
  - 정기 시리즈 강제 X, 데이터 드리븐

### 12.2 커뮤니티 관리
- ✅ **댓글 자동 응답**: ❌ 안 함 (리스크)
- ✅ **좋아요 누르기**: ✅ 자동화
- ✅ **악플 삭제**: ✅ 자동화 (키워드/AI 기반)
- ✅ **커뮤니티 탭**: 추후 확장

### 12.3 운영 시간
- ✅ **업로드 시간**: 최적 시간 전략 적용
  - 채널/주제별 최적 시간 분석
  - YouTube Analytics 기반 동적 조정
  - 초기: 일반적 골든타임 (점심, 저녁, 밤)
  - 성숙기: 데이터 기반 개인화

### 12.4 기타 확정
- ✅ 채널 구조: Config 기반 멀티 채널
- ✅ 초기 채널: 테크 1개
- ✅ 영상 포맷: Shorts → Long-form 확장
- ✅ 영상 스타일: 자막 + 배경 + TTS
- ✅ 페르소나: 채널 = 화자
- ✅ 비용: 초기 최소, 수익 시 재투자
- ✅ 자동화: 점진적 전환
